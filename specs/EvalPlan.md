# Evaluation Specification Document

## 1. Metric Selection

### Primary Metrics (used in eval gate)
- **Accuracy**: Overall classification correctness on golden set
- **Macro F1 Score**: Harmonic mean of per-class F1 scores
  (more robust than average recall for imbalanced sets)

### Secondary Metrics
- **Confusion Matrix**: Identify systematic error patterns
- **Inference Latency**: P95 < 20ms
- **Memory Usage**: < 100MB

### Business Metrics (future)
- **Critical Age Detection Rate**: Recall for youngest group (≤7)
- **Model Stability**: Variance across consecutive test runs

## 2. Data Slices & Segments

### Age-based Slices (defined for golden set evaluation)
1. **Overall**: All 30 golden samples
2. **Young (≤7)**: Age ≤ 7 years
3. **Adolescent (10–13)**: Age 10–13 years
4. **Adult (≥16)**: Age ≥ 16 years

*Note: These slices replace the previous “pup” terminology for consistency.*

### Recording Condition Slices (optional)
5. **High SNR Group**: Signal quality > 0.8
6. **Medium SNR Group**: Signal quality 0.5–0.8
7. **Low SNR Group**: Signal quality < 0.5

## 3. Release Gates

### Hard Gates (Must Pass – evaluated by `pipelines/eval_gate.py`)
| Metric | Threshold | Tolerance | Action on Fail |
|--------|-----------|-----------|----------------|
| Overall Accuracy | 85% | ±3% | Block deployment |
| Macro F1 (overall) | 80% | ±3% | Block deployment |
| Inference Latency | 20ms | +5ms | Optimize model |
| Memory Usage | 100MB | +20MB | Compress model |

### Soft Gates (Warning Level)
| Metric | Warning Threshold | Action |
|--------|-------------------|--------|
| Single Slice Accuracy | < 75% | Investigate data balance; not a blocker |
| Model Size | > 300MB | Consider optimization |
| Training Time | > 60 minutes | Optimize training pipeline |

## 4. Regression Testing

### Golden Dataset Requirements
- **Size**: 30 samples (within 20–50 range)
- **Distribution**: Minimum 2 samples per slice (young, adolescent, adult)
- **Quality**: Simulated with same generator as training data, but different seed
- **Versioning**: File `data/golden_set.csv` – Git versioned

### Regression Rules
1. **Accuracy Regression**: New model cannot be >5% worse than previous version
2. **Slice Protection**: No slice may degrade >10% absolute compared to previous version
3. **Inference Latency**: Cannot increase >10%

### Testing Frequency
- **Every Model Update**: Complete evaluation gate in CI
- **After Data Updates**: Re-evaluate all gates
- **Weekly**: Random sampling from golden set (future automation)
